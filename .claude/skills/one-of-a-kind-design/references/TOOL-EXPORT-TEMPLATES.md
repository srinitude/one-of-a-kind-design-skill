# TOOL-EXPORT-TEMPLATES.md
> Generated from visual-styles-taxonomy v0.1.0 | one-of-a-kind-design skill

Defines the .zip export package structure for each tool category. Every export contains a standard core plus category-specific files, prompt format adaptations, and asset filtering rules.

---

## Standard .zip Core Structure

Every tool export package contains these files regardless of category:

```
{project-name}-{tool-id}-export/
  README.md                  # Import guide: step-by-step instructions for the target tool
  QUALITY-REPORT.md          # Composite quality score, sub-scores, findings, pass/fail
  STYLE-SPEC.md              # Resolved taxonomy style: palette, typography, spacing, texture, motion
  assets/                    # Filtered asset directory (only types the tool accepts)
    images/                  # PNG/WebP/JPEG (if tool accepts images)
    svg/                     # SVG files (if tool accepts svg)
    video/                   # MP4/WebM (if tool accepts video)
    audio/                   # MP3/WAV (if tool accepts audio)
    3d/                      # GLB/FBX/USDZ (if tool accepts three_d)
  prompts/                   # Tool-adapted prompts (if tool accepts prompts)
    generation-prompts.md    # Image/video generation prompts in tool's native format
    editing-prompts.md       # Editing/refinement prompts
  design-tokens/             # Design tokens (if tool accepts design_tokens)
    tokens.json              # Style Dictionary / Tokens Studio format
    variables.css            # CSS custom properties
```

## Asset Filtering Rules

Each tool declares which asset types it accepts. Only accepted types are included in the .zip.

| Tool Category | Images | SVG | Video | Audio | 3D | Code | Design Tokens | Prompts |
|--------------|--------|-----|-------|-------|-----|------|---------------|---------|
| Video Production | yes | no | yes | yes | no | varies | no | yes |
| Image Design | yes | yes | no | no | no | no | yes | yes |
| Web Development | yes | yes | no | no | no | yes | yes | yes |
| AI/ML Platforms | yes | no | yes | no | no | yes | no | yes |
| Code IDEs | yes | yes | no | no | no | yes | yes | no |
| 3D / Spatial | yes | no | no | no | yes | yes | no | no |
| Audio / Music | no | no | no | yes | no | no | no | yes |
| Collaboration | yes | yes | yes | yes | no | no | no | no |
| Presentation | yes | yes | yes | no | no | no | no | no |
| Deployment | yes | yes | no | no | no | yes | yes | no |

### Filtering Logic

```
For each asset in project:
  1. Check tool.accepts[asset.type]
  2. If false, skip the asset entirely
  3. If true, include in the appropriate subdirectory
  4. Apply format conversion if needed:
     - Images: convert to tool-preferred format (PNG for Figma, TIFF for Photoshop print)
     - Video: ensure MP4 H.264 compatibility
     - Audio: WAV for DAWs, MP3 for web tools
     - 3D: GLB for web tools, FBX for game engines, USDZ for Apple
```

## Prompt Format Adaptation per Tool

Raw prompts from prompt-crafter subagents are NEVER exported directly. Every prompt is transformed:

| Tool | Transformation |
|------|---------------|
| Runway | Strip negative prompts, add @reference tags, cap at 1000 chars |
| Midjourney | Append `--ar`, `--style`, `--chaos`, `--weird`, `--stylize` parameters from taxonomy dials |
| ComfyUI | Convert to full workflow JSON node graph |
| Google AI Studio | Format as system_instruction + user message pairs |
| Stability AI | Map style to style_preset parameter ID |
| ElevenLabs | Add SSML delivery markers, map audience_formality to voice presets |
| Figma | Convert design tokens to Tokens Studio JSON format |
| v0 by Vercel | Optimize for shadcn/ui component understanding |
| Cursor | Format as .cursorrules with style conventions |
| Claude Code | Format as CLAUDE.md + .claude/rules/ files |
